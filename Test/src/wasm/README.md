# WASM Integration for Flying Circus Plane Builder

This directory contains the TypeScript bridge layer for the Rust/WASM implementation.

## Architecture

```
HTML/DOM
    ↓
TypeScript UI Layer (components/)
    ↓
TypeScript WASM Bridge (aircraft_bridge.ts, localization.ts)
    ↓
WASM Module (pkg/ - generated by wasm-pack)
    ↓
flyingcircusrust (Rust implementation)
```

## Files

- **localization.ts** - Localization manager using Rust i18n backend
- **aircraft_bridge.ts** - TypeScript wrapper for Rust Aircraft struct
- **components/language_selector.ts** - Language selection UI component
- **components/** - Additional UI components (to be added)

## Building the WASM Module

From the repository root:

```bash
cd flyingcircuswasm
wasm-pack build --target bundler --out-dir ../Test/pkg
```

This generates:
- `../Test/pkg/flyingcircuswasm.js` - JS bindings
- `../Test/pkg/flyingcircuswasm_bg.wasm` - WASM binary
- `../Test/pkg/flyingcircuswasm.d.ts` - TypeScript definitions

## Usage Example

```typescript
import init, { Localization, AircraftWasm } from '../../pkg/flyingcircuswasm';
import { localization } from './wasm/localization';
import { AircraftBridge } from './wasm/aircraft_bridge';

async function initApp() {
    // Initialize WASM
    await init();

    // Initialize localization
    localization.initializeWasm(Localization);

    // Create aircraft bridge
    const bridge = new AircraftBridge();
    await bridge.initialize(init, AircraftWasm);

    // Use the bridge
    const eraBindings = bridge.getEraBindings();
    console.log(eraBindings);
}
```

## Full Localization Flow

When user changes language:
1. TypeScript: `localization.setLocale('de')`
2. WASM: `rust_i18n::set_locale("de")`
3. TypeScript: Locale change callbacks fire
4. Components: Re-fetch UIBindings from WASM
5. WASM: `create_ui_options()` uses `t!()` macro
6. Rust: `t!("Era Section Title")` → "Ära"
7. WASM: Returns bindings with German strings
8. TypeScript: Re-renders UI with new strings
9. User: Sees entire interface in German!
